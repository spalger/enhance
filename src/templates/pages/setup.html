Enhance Setup Guide

Overview

The Problem
In Github repositories, there can sometimes be a lot of noise on the Issues page. Supporters oftentimes submit issues for Enhancements that remain open while others upvote, downvote, and leave comments on these open issues, but contributors don't have a great way to leave these open for the community while stripping them off the Issues page.
The Solution
That's where Enhance comes in. Contributors can close Github feature requests, labeling them with 'enhancement' (or a similar configurable label) and direct supporters to the repository's Enhance voting app.

From here the community can actively monitor all feature requests and discuss with the contributors. The layout is easy to understand, reduces the noise on the issue page, and promotes community involvement.

Enhance uses Firebase for Github authentication and requires Github's public repository scope in order to upvote, downvote, leave comments, and open issues through the API on the user's behalf.

In an effort to support even the largest repositories like Angular without hitting Github's API rate limit, we are utilizing PouchDB to pull down a repositories Issues/Comments and store them in Indexed DB or Local Storage. On page refresh we run a diff on all new issues/comments since the last updated timestamp in the user's persistent data.

Lunr is used to index the persistent data and allow users to easily search for issues by title or issue number.

Messenger is utilized to notify the user of events happening behind the scenes, and easily indicate any required actions or errors that occur. Events are managed through Reflux.

And behind the scenes, WebComponents are utilized for rendering through Deku. The code is written in ES6, and Webpack is utilized in our build process to transpile the code for backwards compatibility.
